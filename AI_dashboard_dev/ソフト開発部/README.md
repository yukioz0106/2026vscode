# ソフト開発部

## ミッション
経営企画部の要件定義・経理部のデータ基盤をもとに、
世界最高水準の経営ダッシュボードを設計・実装・デプロイする。

## 技術スタック（案）

| レイヤ | 採用技術 | 選定理由 |
|-------|---------|---------|
| フロントエンド | React + TypeScript | コンポーネント再利用性・型安全性 |
| チャートライブラリ | Recharts / D3.js | 柔軟なカスタマイズ、日本語フォント対応 |
| バックエンド | Python FastAPI | データ処理との親和性・高速 |
| データ処理 | pandas / polars | 財務CSVの集計・KPI計算 |
| DB | PostgreSQL | 時系列・集計クエリの安定性 |
| インフラ | Docker + Nginx | 開発・本番環境の統一 |
| 認証 | Azure AD SSO | 社内標準認証基盤 |

## 主要タスク

### 1. 設計書の作成（→ `設計書/`）
- システム構成図
- API設計書（エンドポイント一覧）
- DB設計書（テーブル定義）
- フロントエンド画面設計（コンポーネント図）

### 2. ソースコード実装（→ `src/`）
- バックエンドAPI（FastAPI）
- フロントエンド（React）
- データローダー（CSV→DB投入スクリプト）

### 3. インフラ設定（→ `インフラ/`）
- Docker Compose設定
- Nginx設定
- CI/CDパイプライン設定

## 主な成果物ファイル
| ファイル | 内容 |
|---------|------|
| `設計書/system_architecture.md` | システム構成図・技術選定根拠 |
| `設計書/api_spec.md` | REST API仕様書 |
| `設計書/db_schema.md` | DBテーブル定義 |
| `src/backend/` | FastAPI バックエンド |
| `src/frontend/` | React フロントエンド |
| `インフラ/docker-compose.yml` | コンテナ構成 |

## 連携先
- ← 経営企画部：画面要件定義・KPI定義の受領
- ← 経理部：財務CSV・計算ロジックの受領
- → 品質評価部：テスト環境URLの提供・バグ対応
