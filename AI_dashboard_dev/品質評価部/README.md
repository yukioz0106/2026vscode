# 品質評価部

## ミッション
経営ダッシュボードが「正確・使いやすく・信頼できる」ことを第三者視点で保証する。
財務指標の計算正確性から画面UXまで、多角的に品質を評価・改善提案する。

## 品質評価の3軸

### 1. データ正確性（財務KPI検証）
- 算出されたKPI値を開示資料（有価証券報告書・決算短信）と突合
- 経理部の計算ロジックの独立検証
- 異常値・エラー検知テスト

### 2. 機能品質（テスト）（→ `テスト計画/`）
- 機能テスト（全画面・全KPI表示確認）
- 回帰テスト（リリース毎）
- パフォーマンステスト（レスポンスタイム計測）
- セキュリティテスト（認証・アクセス制御）

### 3. ユーザビリティ（→ `ユーザビリティ/`）
- 経営層ユーザーテスト（シナリオベース）
- ダッシュボードの視認性・直感性評価
- モバイル・タブレット対応確認

## 合格基準（品質ゲート）

| 項目 | 合格基準 |
|------|---------|
| KPI計算精度 | 開示値との誤差ゼロ（整数値は完全一致） |
| 画面表示網羅率 | 全KPIカード・全グラフが正常表示 100% |
| レスポンスタイム | 初期表示3秒以内、フィルタ切替1秒以内 |
| バグ残件数 | 重大バグ0件、軽微バグ5件以下 |
| ユーザーテスト合格率 | タスク完了率80%以上 |

## 連携先
- ← ソフト開発部：テスト環境URL・リリースノートの受領
- ← 経理部：KPI計算値の独立検証依頼の受領
- → 経営企画部：要件充足の確認・改善提案
- → ソフト開発部：バグ報告・改善要求の起票
